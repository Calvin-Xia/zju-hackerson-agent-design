# 需求分析文档

## 1. 项目背景

### 1.1 问题描述

教师在教学过程中，需要参考多本教材，但不同教材对同一概念的描述可能存在差异，导致：
1. 知识点重复，信息冗余
2. 概念定义不一致，学生困惑
3. 教学效率低下，需要手动整合

### 1.2 解决方案

开发学科知识整合智能体，自动将多本教材整合为精华版本，实现：
1. 知识点去重和合并
2. 语义对齐和一致性保证
3. 教学完整性保障
4. 精准问答支持

## 2. 子问题分解

### 2.1 知识点粒度定义

**问题**：什么是知识点？如何定义知识点的粒度？

**分析**：
- 知识点是教材中独立的概念、定理、方法或现象
- 粒度应该适中，既不能太细（如单个句子），也不能太粗（如整章）

**解决方案**：
- 以章节为单位进行提取
- 每个知识点包含：名称、定义、分类、章节、频次
- 使用LLM进行智能提取

**示例**：
```
知识点名称：炎症
定义：炎症是具有血管系统的活体组织对各种损伤因子的刺激所发生的防御性反应
分类：核心概念
章节：病理学 第四章
频次：15
```

### 2.2 重复判定标准

**问题**：如何判断两个知识点是否重复？

**分析**：
- 完全重复：定义完全相同
- 语义重复：定义措辞不同，但含义相同
- 部分重复：定义有重叠，但不完全相同

**解决方案**：
- 使用嵌入相似度进行快速筛选（阈值0.7）
- 使用LLM进行精确判断
- 支持三种决策：合并、保留、删除

**判定标准**：
1. 相似度 ≥ 0.9：直接合并
2. 相似度 0.7-0.9：LLM验证
3. 相似度 < 0.7：不重复

### 2.3 教学连贯性保障

**问题**：如何保证整合后的教学逻辑链路不断裂？

**分析**：
- 教学有逻辑顺序，如：细胞 → 组织 → 器官 → 系统
- 删除知识点可能导致逻辑断裂
- 需要保留前置依赖关系

**解决方案**：
- 保留所有前置依赖知识点
- 检查逻辑链路完整性
- 提供知识缺口预警

**验证方法**：
1. 构建知识点依赖图
2. 检查关键路径完整性
3. 人工审核关键节点

### 2.4 压缩比计算方式

**问题**：如何计算压缩比？如何控制在30%以内？

**分析**：
- 压缩比 = 整合后字数 / 原始总字数
- 需要动态调整策略，确保不超过30%

**解决方案**：
- 实时统计字数
- 优先删除低价值内容
- 保留核心知识点

**计算公式**：
```
压缩比 = 整合后总字数 / 原始总字数 × 100%
目标：压缩比 ≤ 30%
```

**优化策略**：
1. 按频次排序，优先保留高频知识点
2. 按定义完整度排序，优先保留完整定义
3. 按教学重要性排序，优先保留核心概念

### 2.5 RAG分块策略选择依据

**问题**：如何选择文档分块策略？

**分析**：
- 分块太大：检索精度低
- 分块太小：上下文不完整
- 需要平衡精度和完整性

**解决方案**：
- 分块大小：500-800字
- 重叠大小：50-100字
- 保留元数据：教材名称、章节、页码

**选择理由**：
1. **500-800字**：足够包含一个完整的知识点描述
2. **50-100字重叠**：避免知识点被截断
3. **元数据保留**：支持引用来源追溯

**对比分析**：

| 分块策略 | 优点 | 缺点 | 适用场景 |
|----------|------|------|----------|
| 按段落 | 自然边界 | 大小不一 | 文档结构清晰 |
| 按固定字数 | 大小一致 | 可能截断句子 | 通用场景 |
| 按语义 | 语义完整 | 实现复杂 | 高精度需求 |

**最终选择**：按固定字数 + 句子边界保护

## 3. 功能需求

### 3.1 教材管理

**需求**：支持多格式教材上传和管理

**功能点**：
1. 支持PDF、Markdown、TXT、DOCX格式
2. 支持批量上传
3. 显示文件信息（名称、大小、格式、状态）
4. 支持删除已上传文件

**验收标准**：
- 上传成功率 ≥ 99%
- 支持200MB以内文件
- 解析状态实时更新

### 3.2 知识图谱构建

**需求**：自动提取知识点和关系，构建可视化图谱

**功能点**：
1. 知识点提取（名称、定义、分类、章节）
2. 关系提取（前置依赖、并列关系、包含关系、应用关系）
3. 图谱可视化（节点大小、颜色、布局）
4. 交互操作（点击、缩放、拖拽）

**验收标准**：
- 提取准确率 ≥ 90%
- 支持1000+节点可视化
- 交互响应时间 < 100ms

### 3.3 跨教材整合

**需求**：将多本教材的知识图谱进行整合

**功能点**：
1. 语义对齐（识别重复知识点）
2. 整合决策（合并、保留、删除）
3. 压缩比控制（不超过30%）
4. 整合统计（决策分布、压缩比、节点变化）

**验收标准**：
- 对齐准确率 ≥ 80%
- 压缩比 ≤ 30%
- 决策理由清晰

### 3.4 RAG问答

**需求**：基于教材内容进行精准问答

**功能点**：
1. 向量索引建立
2. 语义检索（top-5最相关chunk）
3. 答案生成（带引用来源）
4. 引用展开（查看原文）

**验收标准**：
- 检索响应时间 < 3秒
- 引用准确率 ≥ 90%
- 答案相关性 ≥ 85%

### 3.5 多轮对话

**需求**：支持自然语言交互和反馈处理

**功能点**：
1. 对话历史管理
2. 整合决策解释
3. 用户反馈处理
4. 知识图谱实时更新

**验收标准**：
- 对话响应时间 < 5秒
- 反馈处理准确率 ≥ 90%
- 上下文理解准确率 ≥ 85%

## 5. 约束条件

### 5.1 技术约束

1. **前端框架**：React 19 + TypeScript
2. **后端框架**：FastAPI (Python)
3. **LLM服务**：通义千问API
4. **向量数据库**：FAISS
5. **部署平台**：魔搭创空间 / Vercel + Railway

### 5.2 时间约束

- 开发时间：5小时（黑客松）
- 需要快速迭代，优先实现核心功能

### 5.3 资源约束

- **计算资源**：免费CPU，无GPU
- **存储空间**：有限，需要优化
- **API调用**：有成本，需要控制

## 7. 总结

本需求分析详细定义了学科知识整合智能体的功能需求、非功能需求和约束条件。系统核心价值在于：

1. **自动化**：自动完成教材整合，减少人工工作量
2. **智能化**：使用AI技术进行语义对齐和决策生成
3. **可视化**：提供直观的知识图谱和整合结果
4. **可解释**：每个决策都有清晰的理由

系统设计遵循高内聚、低耦合原则，易于扩展和维护。
