# Phase 4: 跨教材知识整合 Spec

## Why
跨教材知识整合是本赛题最核心的技术挑战，需要将多本教材的知识图谱合并，去重提纯，识别不同教材中描述同一概念的知识点，即使措辞不同。整合质量直接影响教学效果和压缩比。

## What Changes
- 实现语义对齐算法，识别不同教材中的重复知识点
- 实现整合决策机制（合并、保留、删除）
- 控制压缩比（整合后保留的内容总字数不超过原始总字数的30%）
- 提供可视化对比功能
- 生成整合决策列表和统计信息

## Impact
- Affected specs: Phase 3（依赖单本教材知识图谱）
- Affected code:
  - 新增: `src/integration/`（整合模块）
  - 新增: `src/alignment/`（对齐算法模块）
  - 修改: `src/kg/`（支持多本教材图谱）
  - 修改: `src/visualization/`（支持整合对比可视化）

## ADDED Requirements

### Requirement: 语义对齐
系统 SHALL 识别不同教材中描述同一概念的知识点，即使措辞不同。

#### Scenario: 文本嵌入对齐
- **WHEN** 对知识点进行语义对齐
- **THEN** 使用文本嵌入（Embedding）计算语义相似度
- **AND** 设定阈值判断是否为同一知识点
- **AND** 支持中文语义理解

#### Scenario: LLM判断对齐
- **WHEN** 对疑似重复的知识点进行判断
- **THEN** 使用LLM判断两个知识点是否等价
- **AND** 提供判断理由
- **AND** 返回置信度分数

#### Scenario: 混合对齐策略（推荐）
- **WHEN** 进行大规模知识点对齐
- **THEN** 先使用文本嵌入快速筛选候选对
- **AND** 再使用LLM精确判断
- **AND** 平衡准确性和成本

### Requirement: 整合决策
系统 SHALL 对识别出的重复知识点做出整合决策。

#### Scenario: 决策类型
- **WHEN** 对知识点进行整合
- **THEN** 支持三种决策类型：
  - merge（合并重复）：将多个重复知识点合并为一个
  - keep（保留唯一）：保留最佳版本
  - remove（删除冗余）：删除明显冗余的知识点
- **AND** 每个决策包含理由和置信度

#### Scenario: 决策数据结构
- **WHEN** 生成整合决策
- **THEN** 输出标准化决策数据：
```json
{
  "decision_id": "merge_001",
  "action": "merge",
  "affected_nodes": ["book01_node_015", "book03_node_032", "book05_node_008"],
  "result_node": "merged_node_001",
  "reason": "三本教材都讲解了'炎症'的概念，保留《病理学》的版本因其描述最系统完整",
  "confidence": 0.92
}
```

#### Scenario: 决策理由生成
- **WHEN** 生成整合决策
- **THEN** 系统对每一项决策给出理由
- **AND** 解释为什么合并/保留/删除
- **AND** 理由清晰易懂

### Requirement: 压缩比控制
系统 SHALL 控制整合后的内容总量，确保不超过原始体量的30%。

#### Scenario: 压缩比计算
- **WHEN** 计算压缩比
- **THEN** 计算原始总字数
- **AND** 计算整合后总字数
- **AND** 压缩比 = 整合后字数 / 原始总字数 × 100%
- **AND** 压缩比不超过30%

#### Scenario: 压缩比统计显示
- **WHEN** 显示整合结果
- **THEN** 在前端展示压缩比统计
- **AND** 显示：原始总字数 → 整合后字数 → 压缩比百分比
- **AND** 数据与实际运行结果一致

#### Scenario: 压缩策略优化
- **WHEN** 压缩比超过30%
- **THEN** 系统自动调整整合策略
- **AND** 优先删除低价值内容
- **AND** 保留核心知识点
- **AND** 确保教学完整性

### Requirement: 可视化对比
系统 SHALL 提供整合前后的知识图谱对比可视化。

#### Scenario: 整合前后对比
- **WHEN** 完成知识图谱整合
- **THEN** 展示整合前后的知识图谱对比
- **AND** 让用户直观看到哪些节点被合并、哪些被删除
- **AND** 使用颜色或大小变化标识变化

#### Scenario: 整合决策可视化
- **WHEN** 显示整合决策
- **THEN** 在知识图谱上高亮显示受影响的节点
- **AND** 显示决策类型（合并/保留/删除）
- **AND** 支持点击查看决策详情

### Requirement: 整合统计
系统 SHALL 提供完整的整合统计信息。

#### Scenario: 统计信息
- **WHEN** 完成整合
- **THEN** 生成整合统计：
  - 原始教材数量
  - 总字数
  - 整合后字数
  - 压缩比
  - 整合决策数量（合并X项、保留X项、删除X项）
  - 知识图谱统计（整合前节点数 → 整合后节点数，关系数变化）

#### Scenario: 统计可视化
- **WHEN** 显示统计信息
- **THEN** 使用图表展示统计数据
- **AND** 支持导出统计报告
- **AND** 数据准确可靠

## MODIFIED Requirements

### Requirement: 知识图谱数据结构增强
现有的知识图谱数据结构需要支持多本教材和整合信息。

#### Scenario: 多教材支持
- **WHEN** 存储多本教材的知识图谱
- **THEN** 节点包含教材来源信息
- **AND** 支持按教材筛选
- **AND** 支持跨教材查询

#### Scenario: 整合状态标记
- **WHEN** 节点被整合
- **THEN** 标记整合状态
- **AND** 记录整合决策ID
- **AND** 保留原始节点引用

## REMOVED Requirements
无

## Technical Constraints
- 语义对齐需要考虑中文语义理解
- LLM调用成本需要控制
- 整合算法需要高效处理大规模知识点
- 压缩比控制需要精确计算
- 可视化对比需要直观清晰

## Dependencies
- Phase 3: 知识图谱构建与可视化（单本教材知识图谱）

## Success Criteria
1. 能准确识别不同教材中的重复知识点（准确率≥80%）
2. 整合决策合理，理由清晰
3. 压缩比不超过30%
4. 可视化对比直观清晰
5. 统计信息准确完整
6. 整合过程高效稳定
